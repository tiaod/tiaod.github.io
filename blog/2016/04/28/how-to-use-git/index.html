#添加远程仓库
记得改成自己的仓库名字。

由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

U参数的作用：

-u, –set-upstream
For every branch that is up to date or successfully pushed, add
upstream (tracking) reference, used by argument-less git-pull(1)
and other commands. For more information, see branch..merge
in git-config(1).

之后的提交就方便了

修改git的默认编辑器

默认使用git commit的时候会自动启动nano编辑器。但是命令行对搜狗输入法好像不友好，并且使用起来不方便，不符合现代的正常使用习惯（要用crtl+x退出，ctrl+o保存文件。很麻烦），所以自定义编辑器为mousepad

使用就是直接修改，保存，关闭，git就会自动提交了。
